Implementation sketch

A concrete control pattern for the Terborg Zorg stack.

The exact table names and node IDs would adapt to the live workspace. The important part is making every automation step explainable, replayable, and safe to pause when the data is not ready.

1. Canonical candidate record

Supabase holds the shared state instead of letting tools drift.

candidate_id: tz_10482
source: meta_ads
ghl_stage: intake_booked
canonical_stage: document_check
consent_logged: true
blocked_reason: null

2. AI extraction is validated

Claude returns typed fields that pass checks before updates.

{
  "work_authorization": "zzp",
  "certificates": ["BHV"],
  "availability": "night shifts",
  "confidence": 0.91,
  "needs_review": false
}

3. Failed handoffs get surfaced

n8n routes bad states to review, not silent retries.

if doc_status_conflict:
  write_dead_letter_row()
  notify_ops(
    candidate_id,
    reason="contract signed before doc check"
  )
else:
  sync_ghl_stage()
